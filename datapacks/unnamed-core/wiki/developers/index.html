Explanation of the main pack

Everything included in the main pack

The pack adds the following features:

  • A player & group ID system
  • A money system
  • A system for blocking the placing of blocks
  • Rotating items in item frames anti-clockwise
  • A creative menu system for custom items.
  • Scoreboards/Tags, including position, sneaking, daytime, etc.
  • Block tags, e.g. beacon base, shulker boxes, etc.
  • Entity tags, e.g. fishes, passive, projectiles, tameable, etc.
  • A fluid tag for water & lava together
  • Item tags, e.g. concrete powder, dyes, terracotta, etc.
  • Function tags, e.g. a tag that runs once every second, a tag that runs on death, a tag that runs on join, etc.

The player ID system

Everything about the player/group ID system

Every player gets assigned on join a unique player ID starting with 1. The first player that joins should be the owner/admin because he will receive admin messages on join. These player IDs must not be changed.

At the start every player is in his own group (with the same ID as his player ID). Group owners can invite the nearest player by running the command /trigger invite_player. The player invited has to accept the invite and his group ID will change to the other players group ID. Everyone except the group owner can leave with /trigger leave_group and the owner can kick the nearest player with /trigger kick_player.

The scoreboards are called player_id and group_id. The scoreboard ud_owner_id can be used to for example store the owner of an entity. Setting this score does nothing. It is only for storing the value and has to be handled by you datapack.

The money system

Everything about the money system

Every player gets on first join a preconfigured amount of money (default is 1000$). The default amount can be changed by running this command /scoreboard players set #startValue money <amount> and replacing <amount> with the new starting value.

Players can check their balance by running /trigger check_money and send money to the nearest player with /trigger pay_money set <amount>.

The money value is stored in the scoreboard money and can be changed by other datapacks with normal scoreboard commands.

The NoPlacing System

The system for blocking the placing of blocks

To block the placing of a block, add the following item NBT to the block: {unnamedDE:{NoPlacing:1}}

Example: /give @s minecraft:dirt{unnamedDE:{NoPlacing:1}}
You won't be able to place this dirt block!

Reverse rotating

How to use reverse rotating

You can rotate the item in an item frame anti-clockwise by holding shift and right-clicking.

The creative menu

Everything about the creative menu

You can view the creative menu by running /function unnamed_core:creative_menu and then select the datapack to view the contents of. Click the number between the square brackets and you will get the corresponding amount of the item.

Create a function with a tellraw command to trigger another function (

tellraw @s [{"text":" ► ","color":"green"},{"text":"unnamed Core","hoverEvent":{"action":"show_text","value":"Click to select"},"clickEvent":{"action":"run_command","value":"/function unnamed_core:creative_menu/menu"}}]
) with tellraw commands with click events to execute a function with tellraw commands for every custom item with click events to give the items you want (
tellraw @s [{"text":"► ["},{"text":"1","color":"green","clickEvent":{"action":"run_command","value":"/give @s diamond"}},{"text":"] [","color":"reset"},{"text":"64","color":"green","clickEvent":{"action":"run_command","value":"/give @s diamond 64"}},{"text":"] ","color":"reset"},{"text":"Diamond"}]
). Add the path of the function to the tag /data/unnamed_core/tags/functions/creative_menu.

Scoreboards/Tags

Description of every scoreboard/tag

Scoreboard names and their description:

Scoreboard Name Description
money The amount of money per player
player_id The player ID per player
group_id The group ID per player
x_pos The X position of the player per player
y_pos The Y position of the player per player
z_pos The Z position of the player per player
x_rot The X rotation of the player per player
y_rot The Y rotation of the player per player
dim The dimension of the player per player
ud_xp The amount of xp levels of the player per player
deaths The amount of deaths of the player per player
ud_deathtime_h The amount of hours since last death of the player per player
ud_deathtime_m The amount of minutest since last death of the player per player
ud_timeplayed_h The amount of hours played per player
ud_timeplayed_m The amount of minutest played per player
ud_const Constant values. Currently:
  • -1: -1
  • 1: 1
  • 2: 2
  • 4: 4
  • 5: 5
  • 8: 8
  • 10: 10
  • 16: 16
  • 32: 32
  • 64: 64
  • 100: 100
  • 128: 128
  • 240: 240
  • 256: 256
  • 1200: 1200
  • 72000: 72000

  • example: /scoreboard players get -1 ud_const
ud_misc Miscellaneous. Currently:
  • day: Minecraft Days
  • real_day: Minecraft Days + 1
  • daytime: Current daytime
  • real_daytime: Current daytime + 6000
  • real_daytime%: (Current daytime + 6000)/240 Percent of the day (noon is 50%)
  • difficulty: Game difficulty (set at load/reload)

Smash Lib

Functionality of Smash Lib

Smash Lib allows you to add events when an entity gets hit with an anvil and when an item gets crushed with a piston from above.

When an entity gets hit with an anvil, the tag #smash_lib:smashed_anvil gets executed.
The tag #smash_lib:smashed_anvil runs when an item gets crushed with a piston.

Example: Horsetastic
